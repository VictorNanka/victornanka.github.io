我和我的家人去红海上的胡尔格达潜水度假。美丽的地方，强烈推荐。第一天，我喝了自来水。可能是，我妈妈叫我不要喝。整个星期，我真的病了–没离开酒店房间。我带着的只是一台笔记本电脑，新安装了 Slackware Linux ，这是我听说过的东西，正想试一试。没有额外的应用程序，只有八张光盘上的东西。出于无奈，我这一周要做的就是熟悉这个陌生的系统。我看了一些网页，玩了一下终端。我记得当时我不知道单点和双点的区别，单点是指当前目录，双点是指以前的目录。

1993年，Windows NT 出来了，它与其他服务器操作系统竞争，但很多开发者都在想，”当我可以用Apache建立一个基于 Linux 的廉价系统时，我为什么要买一个AIX盒子或大型windows盒子？” 关键是，Linux的代码开始渗入到几乎所有在线的东西中。

Steven Vaughan-Nichols:

[00:09:00]

微软意识到，出乎他们意料的是，Linux 实际上已经开始获得了一些业务，不是在桌面上，而是在商业服务器上。结果，他们开始了一场运动，也就是我们喜欢称之为FUD的运动–恐惧、不确定性和怀疑–说：”哦，这个 Linux 的东西，真的没那么好。它不是很可靠。你不能相信它的任何东西。”

这种软宣传式的攻击持续了一段时间。微软也不是唯一一个对 Linux 感到紧张的人。这真的是整个行业与那个奇怪的新家伙的对抗。例如，任何与 UNIX 有利害关系的人都会把 Linux 看作是篡夺者。著名的是，曾经生产过 UNIX 版本的 SCO 集团，为了阻止 Linux 的传播，打了十几年的官司。SCO 最终以失败而破产。与此同时，微软不断地寻找着自己的破绽。他们是一家需要采取行动的公司。只是还不清楚该怎么做。
